# Favi-study — учусь бэкенду на .NET, строя мини-копию Favilonia

Это **учебный полигон**. Здесь я шаг за шагом, руками, собираю крошечную версию
бэкенда Favilonia — чтобы по-настоящему понять, **что**, **зачем** и **почему**
происходит в большом проекте, и подготовиться к работе бэкенд-разработчиком.

> **Главное правило этого репозитория:**
> не двигаюсь дальше, пока не могу объяснить вслух своими словами то, что сделал.
> Скопировать код — не цель. Цель — понять суть.

---

## Зачем это нужно (и почему именно так)

В большой Favilonia много кода написал AI, и при чтении легко терять нить.
Проблема не в проекте, а в том, что **понимание не успевает за кодом**.

Лекарство — обратный порядок: не «прочитать готовое», а **построить маленькое самому**.
Каждую концепцию, которая в Favilonia выглядит сложно (EF Core, JWT, мультитенантность,
тесты), я сначала собираю в игрушечном размере здесь. Потом открываю большой проект —
и **узнаю** её. Перенос знаний — прямой, потому что обе версии используют ровно те же
механизмы .NET 8.

Темы выбраны не случайно: это **ровно те 4 вещи**, по которым проект оценивается как
«крепкий Middle», и **ровно то**, что спрашивают на собеседованиях бэкендеров:
слой данных (EF Core), аутентификация/авторизация (JWT), архитектура и изоляция данных
(мультитенантность), автотесты.

## Как устроен репозиторий

```
Favi-study/
├── README.md          ← ты здесь: что это и зачем
├── ROADMAP.md         ← 6 шагов, маршрут обучения (открой следующим)
├── LEARN.md           ← мой дневник: что я понял на каждом шаге (заполняю сам)
├── glossary.md        ← словарь терминов (DI, middleware, DTO, claim, миграция...)
└── steps/
    ├── step-01-hello-api/      ← ✅ ЭТАЛОН: разобран до последней строки
    ├── step-02-ef-core-crud/   ← задание: код пишу я по подсказкам
    ├── step-03-dto-projection/
    ├── step-04-jwt-auth/
    ├── step-05-multitenancy/
    └── step-06-integration-tests/
```

- **Шаг 1 — готовый образец.** Полностью рабочий, прокомментированный код + длинный
  разбор в README. Его я читаю и ковыряю, чтобы увидеть «как надо».
- **Шаги 2–6 — задания.** В каждом README: цель, объяснение «зачем», пошаговые подсказки,
  чек-лист и ссылка на то же место в большой Favilonia. **Код пишу я сам.** Наставник
  ревьюит и разбирает ошибки.

В конце каждого шага README показывает: «а вот этот же приём в Favilonia, в таком-то
файле» — чтобы связать игрушку с реальным проектом.

## Как пользоваться

1. Прочитай [ROADMAP.md](ROADMAP.md) — увидишь весь маршрут и зачем каждый шаг.
2. Зайди в `steps/step-01-hello-api/`, прочитай README, запусти, поделай задания.
3. Заполни свою секцию в [LEARN.md](LEARN.md) — пара фраз «что я понял».
4. Переходи к Шагу 2 и дальше. Не понял термин — смотри [glossary.md](glossary.md).

Каждый шаг — самостоятельный .NET-проект, запускается из своей папки:

```bash
cd steps/step-01-hello-api
dotnet run
```

## Что нужно установить

- **.NET 8 SDK** (проверь: `dotnet --list-sdks` — должна быть строка `8.0.x`).
- Для Шага 2+ — **PostgreSQL** ИЛИ упрощённо SQLite (решим на месте; для учёбы возьмём
  что проще, чтобы не вязнуть в настройке базы).
- Любой редактор: VS Code (+ расширение C# Dev Kit), Visual Studio или Rider.
